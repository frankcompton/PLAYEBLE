# Reskin Guide

Этот проект — шаблон playable slot-игры.

Основная идея:

- `script.js` — движок игры
- `config.js` — настройки конкретного рескина
- `style.css` — визуальная композиция сцены
- `assets/` — картинки конкретного слота

При обычном рескине лучше НЕ трогать `script.js`.

---

## 1. Что менять при рескине

Обычно меняются только:

1. Фон
2. Логотип
3. Символы
4. Стартовый экран
5. Результаты трёх спинов
6. Цвета интерфейса
7. CTA-текст
8. Offer URL

Всё это находится в `config.js`.

---

## 2. Фон и логотип

В `config.js`:

```js
assets: {
    background: "assets/symbols/background.png",
    logo: "assets/symbols/logo.png",
}

Рекомендуемые размеры:

background: 1080×1920 px, WebP
logo: примерно 1400×500 px или 1600×600 px, PNG/WebP с прозрачностью

Желательно:

фон: 200–500 KB
логотип: 100–300 KB
3. Символы

В config.js:

symbols: {
    s1: "assets/symbols/cherry.webp",
    s2: "assets/symbols/lemon.webp",
    s3: "assets/symbols/bar.webp",
    s4: "assets/symbols/grape.webp",
    s5: "assets/symbols/melon.webp",
    s6: "assets/symbols/orange.webp",
    s7: "assets/symbols/plum.webp",

    coin: "assets/symbols/coin.webp",
    bonus: "assets/symbols/coin2.webp"
}

Для нового слота можно заменить пути на свои картинки.

Рекомендуемые параметры символов:

256×256 px
прозрачный фон
WebP
желательно 20–80 KB на символ

Важно: все символы лучше делать примерно одинаковыми по визуальному размеру внутри холста.

4. Стартовый экран

В config.js:

startScreen: {
    reels: [
        "coin:100.00", "s3", "s1",
        "coin:250.00", "s1", "s1",
        "s7", "s2", "s1"
    ]
}

Порядок символов:

[0] [1] [2]
[3] [4] [5]
[6] [7] [8]

То есть массив читается по строкам сверху вниз.

5. Результаты спинов

В config.js есть блок:

spins: [
    ...
]

Каждый объект — это один клик по кнопке SPIN.

Сейчас логика такая:

Первый спин — проигрыш
Второй спин — маленький выигрыш
Третий спин — jackpot

Пример:

{
    type: "smallWin",
    balance: 350,
    balanceDelay: 300,
    balanceCountDuration: 500,
    winReels: [0],
    reels: [
        "coin:100.00", "s3", "s1",
        "coin:250.00", "s1", "s1",
        "s7", "s2", "s1"
    ]
}
6. Порядок символов в reels

Порядок всегда такой:

[0] [1] [2]
[3] [4] [5]
[6] [7] [8]

Но визуально это 3 рила:

рил 1: [0], [3], [6]
рил 2: [1], [4], [7]
рил 3: [2], [5], [8]

Например:

reels: [
    "coin:350.00", "s2", "s1",
    "coin:250.00", "bonus", "s3",
    "bonus", "s7", "bonus"
]

Это значит:

верхний ряд: coin:350.00 | s2    | s1
средний ряд: coin:250.00 | bonus | s3
нижний ряд: bonus       | s7    | bonus
7. Coin-символы с суммами

Обычный символ задаётся так:

"s1"

Монетка с суммой задаётся так:

"coin:100.00"

Примеры:

"coin:100.00"
"coin:250.00"
"coin:350.00"

Картинка монетки берётся отсюда:

coin: "assets/symbols/coin.webp"

А текст суммы рисуется поверх через код.

8. Jackpot / bonus symbols

В третьем спине есть:

winSymbols: ["bonus"]

Это значит: после остановки символы bonus получат pop/pulse анимацию.

Сам bonus-символ задаётся здесь:

bonus: "assets/symbols/coin2.webp"
9. Интрига на третьем риле

В jackpot-спине есть настройки:

reelDurations: [900, 1200, 4200],
reelFillerCounts: [12, 12, 36],
anticipationReel: 2,
anticipationDelay: 1300,

Что это значит:

первый рил останавливается за 900ms
второй рил останавливается за 1200ms
третий рил крутится 4200ms
третий рил имеет больше filler-символов, поэтому визуально крутится быстрее и дольше
anticipationReel: 2 — подсвечивается третий рил

Индексы рилов:

0 = первый рил
1 = второй рил
2 = третий рил


10. CTA

В config.js:

cta: {
    title: "BIG WIN!",
    amount: "10 350 A$",
    buttonText: "CLAIM BONUS"
}

Можно заменить на:

cta: {
    title: "MEGA WIN!",
    amount: "8 000 $",
    buttonText: "PLAY NOW"
}
11. Offer URL

В config.js:

offer: {
    url: "https://google.com"
}

Для реальной кампании сюда ставится ссылка оффера / трекера.

Во время разработки можно оставить:

offer: {
    url: ""
}

Тогда перехода не будет.

12. Цвета интерфейса

Цвета лежат в блоке:

theme: {
    ...
}

Там можно менять:

цвет верхней плашки
цвет рамки слота
цвет свечения
цвет popup
цвет CTA-кнопки

Пример:

slotBorder: "#4cc8ff",
slotGlow: "rgba(0, 180, 255, 0.7)",
ctaPopupTop: "#233dff",
ctaPopupBottom: "#07075f",
13. Размер сцены

Базовый размер playable задаётся здесь:

scene: {
    baseWidth: 430,
    baseHeight: 760,
    maxScale: 1.15
}

Логика:

вся игра сделана как сцена 430×760
на телефоне и компе она масштабируется целиком
композиция остаётся одинаковой на разных экранах

Обычно это лучше не трогать.

14. Что НЕ трогать при обычном рескине

Обычно не нужно менять:

script.js
основную HTML-структуру
механику рилов
обработчики кнопок
систему gameScaler

Если для нового слота нужно поменять только картинки и сценарий — достаточно config.js и assets.

15. Чеклист перед выгрузкой

Перед загрузкой playable проверить:

игра открывается на телефоне
все 3 спина работают
третий рил крутится дольше
jackpot-символы пульсируют
CTA появляется плавно
кнопка CTA ведёт по offer URL
фон и логотип не слишком тяжёлые
общий вес проекта желательно до 5 MB
